## English

### Prominent Disclosure (Accessibility Service)

**REQUIREMENT:**  
GMT IVR Assistant utilizes the `AccessibilityService` API to provide its core features. This permission is essential for the application to function.

**What we do:**  
Our service observes live call transcripts (such as Bixby Text Call) and on-screen text to identify Interactive Voice Response (IVR) menu options (e.g., "Press 1 for Support").

**Why we do it:**  
The API allows the application to programmatically interact with the call screen and dial pad to navigate these menus on your behalf.

**Privacy & Control:**  
We do not collect, store, or share your personal call data or transcripts. All processing is ephemeral and occurs only during an active call. You can disable this permission at any time in your device settings.

---

### Privacy Policy

**Accessibility Service Usage:**  
The Application requires Android’s Accessibility Service to function. This API is used exclusively to observe and read live call transcripts and on-screen text during phone calls to navigate Interactive Voice Response (IVR) systems. This is the core functionality of the Application.

**Data Processing & Cloud AI:**  
To interpret automated menus, the observed transcript data is transmitted securely to cloud-based Artificial Intelligence models (e.g., Google Gemini). This data is processed in real-time solely to navigate the active call. It is ephemeral; we do not permanently store, log, or sell any transcript data or Personally Identifiable Information (PII). All processed data is deleted from our active memory immediately upon the termination of the call session.

**Third-Party Retention:**  
While the Developer does not store your data, Users acknowledge that the Application cannot control the transient data processing logs or data retention policies of the third-party AI model providers used to facilitate this service.

## Español

### Divulgación Destacada (Servicio de Accesibilidad)

**REQUISITO:**  
GMT IVR Assistant utiliza la API `AccessibilityService` de Android para proporcionar sus funciones principales. Este permiso es esencial para que la aplicación funcione.

**Qué hacemos:**  
Nuestro servicio observa las transcripciones de llamadas en vivo (como Bixby Text Call) y el texto en pantalla para identificar las opciones del menú de Respuesta de Voz Interactiva (IVR) (por ejemplo, "Presione 1 para Soporte").

**Por qué lo hacemos:**  
La API permite que la aplicación interactúe programáticamente con la pantalla de llamada y el teclado para navegar por estos menús en su nombre.

**Privacidad y Control:**

### Política de Privacidad

**Uso del Servicio de Accesibilidad (Accessibility Service):**  
La Aplicación requiere el Servicio de Accesibilidad de Android para funcionar. Esta API se utiliza exclusivamente para observar y leer las transcripciones de llamadas en vivo y el texto en pantalla durante las llamadas telefónicas con el fin de navegar por los sistemas de Respuesta de Voz Interactiva (IVR). Esta es la función principal de la Aplicación.

**Procesamiento de Datos e IA en la Nube:**  
Para interpretar los menús automatizados, los datos de la transcripción observada se transmiten de forma segura a modelos de Inteligencia Artificial basados en la nube (por ejemplo, Google Gemini). Estos datos se procesan en tiempo real únicamente para navegar por la llamada activa. Son efímeros; no almacenamos, registramos ni vendemos de forma permanente ningún dato de transcripción ni Información de Identificación Personal (PII). Todos los datos procesados se eliminan de nuestra memoria activa inmediatamente al finalizar la sesión de la llamada.

**Retención por Terceros:**  
Aunque el Desarrollador no almacena sus datos, los Usuarios reconocen que la Aplicación no puede controlar los registros transitorios de procesamiento de datos o las políticas de retención de datos de los proveedores de modelos de IA de terceros utilizados para facilitar este servicio.
